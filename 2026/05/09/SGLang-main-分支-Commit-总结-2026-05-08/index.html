SGLang main 分支 Commit 总结 2026-05-08

Created2026-05-09|Updated2026-05-09|技术笔记SGLang
|Post Views:

本文档汇总了 SGLang 主仓库 sgl-project/sglang 在 2026 年 5 月 8 日（UTC+8，00:00–23:59） 期间合并到 main 分支的全部 commit，共计 37 个。以下按模块分类，对用户重点关注的新模型支持、性能优化、Bug Fix、server_args.py 新增参数以及新增环境变量进行了提炼与表格化梳理。

一、新模型支持

commit message 总结 PR链接
Deepseek V4 (#23882) 完整引入 DeepSeek-V4 模型支持，新增 dsv4 专属 Attention Backend 及 JIT Kernel 实现（c128/c4 系列），并加入 compressed 别名兼容旧配置。 PR #23882
[Gemma 4] Adding MTP support (#24436) 为 Gemma 4 模型新增 Multi-Token Prediction (MTP) 支持，引入 Frozen KV MTP Worker 与 CUDA Graph Runner。 PR #24436
[NPU]Support model Trinity-mini for Npu, accuracy 90% (#18172) 在昇腾 NPU 上新增对 Trinity-mini 模型的支持，实现约 90% 精度。 PR #18172
[sgl-kernel/cpu] support w8a8 int8 model for arm cpu (#16045) 在 ARM64 CPU 后端上新增 w8a8 INT8 量化模型支持，包含专用 GEMM 与 MoE 算子。 PR #16045
[AMD] Support fp8 MLA for diffusion model (#20319) 在 ROCm 平台为扩散模型（DiT）新增 FP8 MLA Attention 支持。 PR #20319
:sparkles: [diffusion][npu][quant] Add MXFP8 quantization support for Wan2.2 Diffusion on Ascend NPU (#20922) 为昇腾 NPU 上的 Wan2.2 扩散模型新增 MXFP8 量化方案与模型加载支持。 PR #20922

二、性能优化

commit message 总结 PR链接
[diffusion] optimize: optimize frame returns path (#24616) 优化扩散模型帧数据返回路径，引入 IPC Array 机制以减少 GPU-CPU 传输开销。 PR #24616
[diffusion] optimize: precompute LTX2 guidance perturbation states (#24494) 对 LTX2 模型的 guidance perturbation 状态进行预计算，减少推理时重复计算。 PR #24494
Cache empty MatchResult in RadixCache (#24470) 在 RadixCache 中缓存空的 MatchResult，避免重复创建对象，降低内存分配与 GC 压力。 PR #24470
Turn on JIT custom AR implementation by default (#24363) 默认启用 JIT 编译的 Custom All-Reduce (AR) 实现，替代原有两阶段 AllReduce。 PR #24363
Enable flashinfer::trtllm_allreduce_fusion with PDL (#23765) 启用 flashinfer 的 TRT-LLM AllReduce Fusion 算子，并与 PDL (Persistent Device Loop) 结合以提升通信效率。 PR #23765
[AMD] Replace naive triton RMSNorm with aiter RMSNorm for diffusion model (#24360) 在 ROCm 扩散模型中用 aiter 高性能 RMSNorm 替换原生 Triton 实现。 PR #24360
[AMD][diffusion] Temporal-unfolded batched Conv2D for ROCm VAE decode (#22971) 为 ROCm VAE decode 引入时间维度展开的批量 Conv2D 替代 CausalConv3d，提升 ROCm 上扩散模型解码效率。 PR #22971
fix(aiter): drop FP8 KV upcast; use native FP8 path in paged_attention (#24129) 移除 FP8 KV Cache 的上转 (upcast) 操作，改用原生 FP8 计算路径，降低显存带宽并提升 attention 效率。 PR #24129
[NPU][diffusion] add selectable parallel VAE decode strategies (#23248) 为昇腾 NPU 扩散模型提供可选择的并行 VAE decode 策略，优化大分辨率生成吞吐。 PR #23248
[MUSA][18/N] Add MUSA-optimized kernel implementations for hot ops (#23255) 为摩尔线程 MUSA 平台新增一系列热点算子（fused_add_rms_norm、MoE GEMV+SwiGLU、采样等）的高性能内核。 PR #23255
[SPEC V2][2/N] feat: adaptive spec support spec v2 (#23336) 扩展自适应投机采样 (Adaptive Speculation) 以支持 Speculative Decoding V2，新增 Eagle Worker V2 及多层 Eagle 实现。 PR #23336

三、Bug Fix

commit message 总结 PR链接
[diffusion] fix: fix FA3 varlen out argument handling (#24688) 修复 FlashAttention-3 varlen 模式下 out 参数传递错误的问题。 PR #24688
[fix] /pause_generation and /continue_generation wrong for –tokenizer-worker-num > 1 (#24462) 修复当 --tokenizer-worker-num 大于 1 时，/pause_generation 与 /continue_generation API 行为异常的问题。 PR #24462
Fix stuck when enabling MTP on DSA models (#24635) 修复在 DeepSeek DSA 模型上启用 MTP (Multi-Token Prediction) 时可能卡死的 bug。 PR #24635
fix(http): apply SGLANG_TIMEOUT_KEEP_ALIVE in common.py (#24323) 修复 SGLANG_TIMEOUT_KEEP_ALIVE 环境变量在 common.py 中未被正确应用的问题。 PR #24323
fix(mimo_v2): auto-disable multimodal when vision/audio configs are absent (#24652) 修复 MiMo v2 模型在缺少 vision/audio 配置时未自动关闭多模态功能的问题。 PR #24652
fix(openai): map reasoning.enabled to thinking AND enable_thinking (#23951) 修复 OpenAI API 兼容层中 reasoning.enabled 未同时映射到 thinking 和 enable_thinking 的问题。 PR #23951
fix(function_call): handle Kimi-K2.5 bare numeric tool call IDs (#23950) 修复 Function Call 解析器无法处理 Kimi-K2.5 裸数字格式 tool call ID 的问题。 PR #23950
fix(router): configure HTTP client connection settings (#24330) 修复 SGLang Model Gateway (Router) 中 HTTP 客户端连接配置未正确生效的问题，并补充相关配置项。 PR #24330
[AMD] Cherry-pick aiter commit for mhc_pre fix (#24665) 从 aiter 仓库 cherry-pick 修复 mhc_pre 相关问题的 commit。 PR #24665

四、启动参数变更（server_args.py）

commit message 总结 PR链接
Deepseek V4 (#23882) 新增 dsv4 Attention Backend 选项，并加入 compressed 作为其向下兼容别名；新增 _validate_dsv4 与 _set_default_nsa_backends 相关逻辑。 PR #23882
[Gemma 4] Adding MTP support (#24436) 新增 Frozen KV MTP 相关启动参数（含 --frozen-kv-mtp-draft-len、-num-mtp-layers 等），扩展 MTP 配置体系。 PR #24436
feat(constrained): two-phase reasoning grammar + –enable-strict-thinking (#23953) 新增 --enable-strict-thinking 参数，用于在 reasoning phase 中启用严格的 token 过滤。 PR #23953
feat(reasoning): auto-detect reasoning/tool-call parser from chat template (#23952) --reasoning-parser 与 --tool-call-parser 新增 auto 选项，支持从 chat template 自动检测解析器。 PR #23952
Add registry for custom speculative algorithms (#23991) --speculative-algorithm 不再限制为内置枚举值，支持通过 SpeculativeAlgorithm.register 注册自定义算法名称。 PR #23991
:sparkles: [diffusion][npu][quant] Add MXFP8 quantization support for Wan2.2 Diffusion on Ascend NPU (#20922) 扩散模型 runtime 的 server_args.py 新增 --quantization 参数，支持显式指定 mxfp8、fp8、modelslim 等量化方式。 PR #20922
Add Arm64 CPU Phase 1A CI bootstrap (#22123) CPU 后端默认 Attention Backend 逻辑调整：在 ARM64 主机上默认使用 torch_native，x86 上保持 intel_amx。 PR #22123

五、新增/变更的环境变量

commit message 总结 PR链接
[AMD] Intro SGLANG_DIFFUSION_AITER_FP8_ATTN (#24677) 新增环境变量 SGLANG_DIFFUSION_AITER_FP8_ATTN，用于在 ROCm 扩散模型中显式控制是否启用 FP8 Attention（替代旧变量 SGLANG_AITER_FP8_ATTN）。 PR #24677
[AMD][diffusion] Temporal-unfolded batched Conv2D for ROCm VAE decode (#22971) 新增两个 ROCm 扩散模型环境变量：SGLANG_USE_ROCM_VAE_CONV2D 与 SGLANG_USE_ROCM_VAE_CONV2D_BF16，分别控制是否用 Conv2D 替换 CausalConv3d 以及是否使用 BF16 精度。 PR #22971
Turn on JIT custom AR implementation by default (#24363) 变更 SGLANG_OPT_USE_CUSTOM_ALL_REDUCE_V2 的默认值从 False 为 True，默认启用 JIT Custom All-Reduce。 PR #24363
