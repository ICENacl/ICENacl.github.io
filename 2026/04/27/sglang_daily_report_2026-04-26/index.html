一、总体概述

昨日 Sglang 主仓库共有 17 次提交，主要涉及以下几个方面：

  1. Bug 修复: 修复了 Qwen3 MoE 在 DP attention + EP + reduce_scatterv 场景下的双重 reduce 问题，并推广到多个 MoE 模型。修复了 NPU 的 support_triton bug 和 parallel_state 中的拼写错误。
  2. 性能优化: 废弃了 act_and_mul_triton，将 filter_expert 整合到 JIT silu/gelu_and_mul 中，优化 MoE kernel 执行效率。
  3. Diffusion 模型改进: 重构 timestep scheduler，使其成为 request-local，提升了多请求并发时的隔离性。
  4. CI/CD 改进: 新增了 DeepSeek-V4 Docker 发布工作流，优化了 PyPI nightly 和 kernel wheel 构建流程。
  5. 文档更新: 补充了 DeepSeek-V4 在 H200 和 GB200 平台的部署指南及验证参数。

四、总结与影响分析

4.1 关键改进

  • MoE 并行稳定性: 连续 3 个 PR (#23731, #23734, #23732) 系统性修复了 DP attention + Expert Parallel + reduce_scatterv 组合下的 reduce 问题。覆盖了 13 个 MoE 模型，显著提升了多机多卡训练/推理的正确性。
  • MoE Kernel 优化: 通过废弃 act_and_mul_triton 并将 filter_expert 整合到 JIT kernel，预计可降低 MoE 激活函数的 kernel 启动开销。
  • Diffusion 并发安全: timestep scheduler request-local 化消除了多请求间的状态竞争，提升了 diffusion 模型的并发服务能力。

4.2 对用户的建议

  • 使用 MoE 模型（特别是 Qwen3MoE、DeepSeekV2 等）且开启了 DP attention + EP 的用户，建议尽快更新以获取 reduce 修复。
  • 使用 diffusion 模型的用户将受益于更好的并发隔离，无需修改配置。
  • 使用 NPU 设备的用户可获得 support_triton 判断的修复。
